# 플레이스 분석 — 기능 정리 (운영 가이드)

와벨리 사장님용 네이버 플레이스 순위·지수 분석 기능. adlog/adpin 같은 유료툴을 직접 대체.

## 무엇을 하나
- **노출 키워드 자동 발굴**: 키워드를 안 넣어도, 매장 카테고리·주소(동)·브랜드명·상세설명에서 후보를 만들어 네이버 검색광고로 확장한 뒤, **300위까지** 훑어 그 매장이 노출되는 키워드를 자동으로 찾는다. (adlog식)
- **키워드별 실제 순위·저장수·리뷰** 수집 (네이버 직접 수집 → adpin과 일치).
- **N지수 추정** (N1 유사도·N2 관련성·N3 랭킹) — 아래 참고.
- **종합 대시보드**: 노출점수·최고/평균순위·노출 키워드수·TOP3/10·순위분포·급등급락·일자별 추이.
- **관리자 전용 "다른 매장 분석"**: 아무 플레이스 ID나 추가해 경쟁·관심 매장 분석(공개엔 노출 안 됨).

## 구조 (수집은 집 PC, 표시는 앱)
네이버 스크래핑은 클라우드에서 차단되므로 **집 PC 수집기(Playwright)**가 담당하고, 결과만 Supabase에 적재 → 앱이 RLS로 본인 것만 표시.
- 앱: `workmate/src/app/place-rank.tsx` (홈 빠른메뉴 📈 플레이스 분석)
- 수집기: `260616/place_collector/place_collector/`
  - `place_discover.py` — 매장별 키워드 발굴(300위)
  - `collector/wavely_sync.py` — 분석·Supabase 적재 (`_analyze_store`)
  - `nestimate.py` — N지수 추정식
  - `scheduler/analysis_worker.py` — 온디맨드 분석 워커(상시 폴링)
  - `wavely_index.py` — 춘천 키워드 색인(일일 배치용)

1. **온디맨드 워커 상시 실행**: `wavely_worker.bat` (또는 작업 스케줄러 `PlaceRank\온디맨드워커`). 앱에서 [분석 시작]/관리자(+) 누르면 이 워커가 처리.

## N지수 = 추정치 (중요)
adlog/adpin의 N1/N2/N3는 **그들 고유 모델의 서버계산값(정확공식 비공개, 네이버도 미노출)** 임을 규명함(→ `N지수-규명.md`). 따라서 와벨리는 **추정**한다:
```
N1 ≈ 0.70958 − 0.11139·log₁₀(전체업체수) + 0.0205·log₁₀(검색량)   (R²0.75, ±0.03)
N3 ≈ 0.25756 + 0.27668·N1                                        (R²0.998)
N2 ≈ 0.35267 + 0.01124·log₁₀(방문리뷰)
```
- 앱에 **"추정치·참고용"**으로 명시. 순위·저장·리뷰는 실측이라 adpin과 일치.
- 정밀화: adpin 샘플 더 모아 `nformula.py`에 추가 → 재적합.

## 노트북(수집기) 꺼졌을 때
네이버가 클라우드 IP를 차단해 수집기는 반드시 집 IP(노트북)에서 돈다. 노트북이 꺼지면:
- **앱·데이터(Supabase/웹)는 클라우드라 그대로 살아있음** — 기존 순위·N지수 항상 조회 가능.
- 새 분석 요청은 `place_analysis_requests`에 **pending으로 안전 저장(유실 없음)** → 노트북 켜지면 **오래된 순서대로 자동 처리**.
- 워커가 30초마다 `worker_heartbeat.last_seen` 갱신(SQL 14) → 앱이 last_seen 3분+ 오래되면 **"요청 저장됨, 서버 켜지면 자동 분석"**으로 친절히 안내(혼란스러운 타임아웃 대신).
- **진짜 24시간 무중단 수집**을 원하면: ①항상 켜진 집 기기(미니PC/라즈베리파이)에 수집기 이식, 또는 ②클라우드 수집기+주거용 프록시(유료). + 노트북 절전 끄기·전원 연결.

## 알려진 한계
- N은 추정치(±0.03~0.05). 브랜드/단독노출 키워드(업체수<30)는 N 추정 제외.
- 온디맨드 분석은 300위 깊게 훑어 수 분 소요(앱 폴링 6분, 미완료 시 화면 재진입으로 표시).
- 워커 브라우저가 죽으면 프로세스 재시작 필요(작업 스케줄러가 프로세스 크래시는 자동 재시작).
