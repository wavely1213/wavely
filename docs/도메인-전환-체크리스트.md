# 도메인 전환 체크리스트 (커스텀 도메인 살 때 한 번에)

현재 운영 도메인: **`wavely-nine.vercel.app`** (Vercel 무료).
커스텀 도메인(예: `wavely.com`) 구매 후, 아래를 **새 도메인 `NEWDOMAIN`** 으로 바꾸면 됨.
대부분 코드는 동적(`location.origin`)이라 자동. 바꿀 곳은 ① 외부 콘솔 ② 하드코딩 1곳.

---

## 1. Vercel — 도메인 연결 (사장님)
- 프로젝트 `wavely` → Settings → **Domains** → `NEWDOMAIN` 추가
- 등록업체 DNS에 Vercel이 주는 레코드 입력 (또는 네임서버 변경) → SSL 자동
- (선택) `NEWDOMAIN` 을 Primary로

## 2. Supabase Auth — redirect/site_url (제가 토큰으로 가능)
Authentication → URL Configuration (또는 Management API):
- **Site URL**: `https://wavely-nine.vercel.app` → `https://NEWDOMAIN`
- **Redirect URLs(uri_allow_list)**: `https://NEWDOMAIN`, `https://NEWDOMAIN/**` 추가
  - (현재값: localhost:8081, wavely://, wavely-nine.vercel.app/**, wavely-*-wavely1213s-projects.vercel.app/**)
- 이거 안 하면 로그인 후 리다이렉트/이메일 인증 링크가 깨짐.

## 3. 네이버 로그인 (developers.naver.com — 사장님 계정)
내 애플리케이션 → 해당 앱 → API 설정:
- **서비스 URL / PC 웹**: `https://NEWDOMAIN` 추가
- Callback URL은 **Supabase 함수 주소**(`…supabase.co/functions/v1/naver-auth/callback`)라 **안 바뀜** — 그대로 둠.

## 4. 네이버 지도 / NCP (console.ncloud.com — 사장님 계정)
Services → Maps(또는 Application) → 인증 정보 → **Web 서비스 URL**:
- `https://NEWDOMAIN` 추가 (현재 wavely-nine 등록돼 있을 것)
- 안 하면 explore·매장등록 지도가 안 뜸.

## 5. PortOne 결제 (admin.portone.io — 사장님 계정)
상점/결제연동 설정 → **허용 도메인 / 리디렉션 URL**:
- `https://NEWDOMAIN` 추가
- 안 하면 결제·빌링키 창이 "Failed to fetch".

## 6. 코드 하드코딩 1곳 (제가 수정)
- `src/components/ReactionBar.tsx:68` 의 네이티브 공유링크 fallback `https://wavely.app` → `https://NEWDOMAIN`
  - (웹은 location.origin이라 자동. 이건 앱(네이티브) 공유링크용)
- 그 외 코드는 전부 동적 → 수정 불필요.

## 7. 참고 (안 바꿔도 되는 것)
- Vercel **환경변수**: 도메인 안 들어있음(키만) → 변경 불필요.
- 네이버 로그인 **Callback**(supabase 함수): 고정 → 변경 불필요.
- PortOne **API secret / store id**: 변경 불필요.
