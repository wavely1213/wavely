## 결정 완료
1. ✅ **비로그인 마켓/구인구직 접근 정책** — **결정: 목록은 공개, 상세는 로그인 게이트** (당근식 유입 + 보호 절충)
   - DB: `market_items` 본문(body), `jobs` 본문(body)·연락처(contact)를 anon 권한에서 제외(column-level GRANT). 비로그인 직접 호출 시 401 검증 완료.
   - 앱: 목록(`market.tsx`/`jobs.tsx`)은 비로그인도 열람. 상세(`market/[id].tsx`/`jobs/[id].tsx`)는 비로그인 시 본문·판매자/작성자·연락처·채팅 숨기고 🔒 로그인 게이트 노출. 로그인 시 전체 노출.
   - 검증: 두 상세 라우트 번들 정상 컴파일, 콘솔 에러 0.

## ✅ 자율 작업 진행 로그 (참고)
- 내활동에 '내 판매·내 공고' 탭 추가
- 내정보에 '동네 생활'(중고거래·구인구직·인기글·키워드) 섹션 추가
- **버그 2개 자율 발견·수정**:
  1. my.tsx else-if 체인 문법오류 → 번들 전체 깨졌던 것 수정 (커밋 amend)
  2. market_items/jobs의 seller·author FK가 auth.users 참조라 profiles 임베드(상세화면) 실패 → **profiles 참조로 변경**(DB, 라이브 반영). 상세화면 복구.
- 신규 화면 전수 검증: 마켓(목록·등록·상세), 구인구직(목록·등록·상세), 인기글, 키워드, 내활동, 빠른메뉴 — 실데이터로 정상.
