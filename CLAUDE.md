## 와벨리 디자인 규칙 (필수 준수 — 변형 금지)

이 저장소의 UI를 만들거나 고칠 때는 **재해석하지 말고 그대로 재현**한다. (디자인 핸드오프: Downloads/_proto_* 의 프로토타입 HTML)

### 단일 진실 소스
- 색·라운드·그림자·간격·폰트·배지색·광고3종은 **`src/constants/wavely.tokens.ts`(= theme.ts 재수출)** 값만 사용한다.
  새 색/간격을 발명하지 않는다. 하드코딩 hex 금지 — 항상 토큰을 import 한다.
- 디자인 레퍼런스는 핸드오프의 프로토타입 HTML이다. 그것이 **픽셀 스펙**이다.

### 하지 말 것 (변형 금지)
- 색·폰트(Pretendard 고정)·라운드·간격·그림자 임의 변경
- 레이아웃 구조·네비게이션·화면 순서 재설계, 컴포넌트/라벨/문구 이름 변경, 섹션 추가·삭제, "개선" 명목 변경
- 광고 3종(배너/인피드/플레이스)·역할 4종(손님/알바/직장인/사장님)·게시판(자유/홍보/사장님/직장인) 구성 변경

### 반드시 할 것
- "리스트=클린(구분선·그림자X), 카드=소프트(라운드·그림자O)" 블렌드 유지
- 모르거나 빠진 값은 **추측 말고 질문**
- 한 번에 한 화면씩 프로토타입과 1:1, 완료 후 다른 점만 보고
- 앱·소비자웹·관리자웹은 **같은 Supabase** 공유(새 DB 금지), 마케팅만 별도 정적
