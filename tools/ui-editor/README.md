# 와벨리 UI 에디터

색상·크기·로고를 **마우스로 바꾸고 → 테스트 앱으로 검수 → 실제 앱에 적용**하는 비주얼 편집 도구예요. 코딩 필요 없음.

## 준비

1. **앱 미리보기**가 켜져 있어야 해요 (`npm run web` / Expo 개발 서버, 보통 http://localhost:8081).
2. `tools/ui-editor/start.bat` 더블클릭 → 브라우저에서 에디터가 열려요. (Chrome 또는 Edge)
3. 에디터에서 **[📂 설정 폴더 열기]** → `workmate/src/constants` **폴더**를 선택. (한 번만)

## 2단계 워크플로우

### ① 검수 (테스트 앱에서 확인)
1. 왼쪽에서 색·크기·로고를 바꿔요. 오른쪽 **미리보기**가 실시간으로 변해요.
2. **[🧪 테스트 저장]** → 바뀐 값이 **검수용 파일**(`ui-config.draft.json`)에 저장돼요. (실서비스엔 영향 없음)
3. **[🔍 테스트 앱 열기]** → 실제 앱이 **검수 모드**(상단에 주황 띠 표시)로 열려요. 모든 화면을 돌아다니며 확인하세요.
   - 값을 또 바꿨으면 **[🧪 테스트 저장]** 다시 누르고 테스트 앱을 새로고침하면 돼요.

### ② 적용 (실제 앱에 반영)
4. 검수가 끝나 만족스러우면 **[🚀 실제 적용]** → 그 값이 **실서비스 파일**(`ui-config.json`)로 승격돼요. 실제 사용자에게 보이는 디자인이 바뀝니다.
5. **[✕ 검수 종료]** → 앱을 평소(실서비스) 모드로 되돌려요. (앱 상단 주황 띠의 ‘검수 종료’ 버튼으로도 가능)

> **검수 모드 vs 실서비스**
> - `?ui=test` → 검수용(draft) 값으로 보기 (상단 주황 띠)
> - `?ui=live` → 실서비스(live) 값으로 보기
> - 한 번 켜면 브라우저에 기억돼요. 네이티브 앱은 항상 실서비스 값만 사용해요.

> Chrome/Edge가 아니면 **[⬇️ JSON 내려받기]** 받아서 `src/constants/ui-config.json`에 덮어쓰면 돼요.

## 무엇을 바꿀 수 있나

- **색상**: 라이트/다크 각각 메인·배경·카드·테두리·글자·뱃지 등 전체 팔레트
- **크기·간격**: 모서리 둥글기, 글자 크기 배율, 배너 높이, 게시글 세로여백·썸네일 크기
- **로고**: 게시판 상단에 들어가는 앱 로고 이미지

값은 `src/constants/ui-config.json`(실서비스) / `ui-config.draft.json`(검수용) 두 파일에 저장돼요.
더 세밀한 항목을 에디터에 추가하고 싶으면 알려주세요.
